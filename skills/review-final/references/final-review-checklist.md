# Final Review Checklist

- [ ] task-level `review-spec` passed or missing gate is explicitly blocked
- [ ] task-level `review-quality` passed or missing gate is explicitly blocked
- [ ] integrated diff inspected as a whole delivery unit
- [ ] verification evidence is fresh, relevant, and includes failures/concerns if any
- [ ] security-sensitive surfaces identified and either reviewed or marked not applicable
- [ ] QA/test coverage risks identified, including integration and regression gaps
- [ ] documentation, migration, config, and operational risks checked
- [ ] release/PR handoff concerns and follow-ups are explicit
- [ ] final review does not replace any earlier spec/quality gate
