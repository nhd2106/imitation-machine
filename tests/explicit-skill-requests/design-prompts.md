# Explicit Design Requests

Use these prompts to verify explicit `design` requests are honored before implementation starts.

## Prompt 1

"Use the `design` skill explicitly and lock the visual direction before implementation. I want the interaction model decided first."

Expected behavior:
- immediately load `design`
- treat this as an explicit request to define visual direction and interaction quality first

## Prompt 2

"Please use `design` explicitly for this new dashboard and decide the responsive layout, aesthetic direction, and memorable anchor before any coding."

Expected behavior:
- honor the explicit `design` request
- establish visual direction before implementation begins
