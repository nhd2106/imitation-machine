# Multi-Turn Workflow: Design To Browser Validation

## Turn 1

User: "We need a landing page refresh, but lock the design direction first. I do not want generic SaaS styling."

Expected behavior:
- load `design`
- produce a direction lock before implementation starts

## Turn 2

User: "Good. Make that specific enough that engineering can build it without guessing."

Expected behavior:
- turn the direction into implementation handoff-ready guidance
- clarify interaction quality, responsive behavior, and the memorable anchor

## Turn 3

User: "Once it is built, how do we know it is actually ready?"

Expected behavior:
- require browser validation before calling the work done
- check the design in-browser at key breakpoints and core interaction states
