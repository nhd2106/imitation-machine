# Multi-Turn Workflow: Brainstorm To Plan

## Turn 1

User: "Let's build a team activity feed for our admin app."

Expected behavior:
- load `brainstorm`
- ask clarifying questions before planning

## Turn 2

User: "Success means admins can filter by actor and event type. Out of scope: notifications."

Expected behavior:
- continue brainstorming until design intent is clear

## Turn 3

User: "Looks right. Now write the implementation plan."

Expected behavior:
- transition from `brainstorm` to `plan`
- produce atomic executable tasks
